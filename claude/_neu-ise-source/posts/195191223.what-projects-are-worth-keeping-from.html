The end of the semester is when digital chaos becomes visible. Your Downloads folder has 247 files. Your desktop looks like a tornado hit it. You have three different “Final_Project” folders and you’re not sure which one you actually submitted. Your GitHub has half-finished class assignments sitting next to the one project you’re actually proud of, and a recruiter just looked at your profile.

Here’s what happens if you don’t deal with this now: You’ll spend 20 minutes hunting for your best project when you need it for a job application. You’ll redownload the same PDF four times because you can’t remember where you saved it. You’ll lose work you’re actually proud of in a folder called “Stuff” or “Old Files.” And six months from now, you’ll have twice as much chaos and half the memory of what anything was for.

This is fixable. Not someday. This weekend. Two hours, six steps, and you’re done.

Step 1: The Four-Folder System (15 minutes)

Create four folders. That’s it. Not twenty. Four.

Projects - Things with deadlines. Your capstone. Your co-op search. The side project you’re actually working on. If it’s not active right now, it doesn’t go here.

Areas - Ongoing stuff with no end date. “Finances.” “Job Search Materials.” “Recipes.” Things you’ll need to update regularly but that never really finish.

Resources - Things you might need later. That data structures cheat sheet. The Docker tutorial you bookmarked. Research papers for topics you’re interested in but not currently working on.

Archives - Everything from this semester that’s over. Completed coursework. Finished group projects. The classes you’re never taking again.

Open your file manager right now. Make these four folders. Everything you have goes into one of them. If you can’t decide, it goes in Archives. You can always pull it back out.

Step 2: Triage Your Semester (30 minutes)

Go through your Documents, Desktop, and Downloads folders. Every file gets sorted into one of three piles:

Keep for Portfolio - The project you’re proud of. The paper where you actually understood the topic. The group project where you did most of the heavy lifting and it turned out well. These go in Projects or Resources depending on whether you’re actively updating them.

Keep for Reference - Lecture notes from classes in your major. The study guide that actually helped. Lab assignments with code you might reuse. Syllabi from classes you might TA. These go in Resources or Archives.

Delete - Rough drafts. Duplicate downloads. PDFs you never opened. Screenshots from group chats about meeting times. Software installers you already installed. Old resume versions that aren’t your current one. Every file named “Untitled” or “Document1.” Everything goes.

Be honest. If you haven’t opened it since you downloaded it, you’re not going to. If it’s a rough draft and you have the final version, the rough draft is gone. If it’s a group project where someone else did all the work and you just edited formatting, it’s not portfolio material.

One decision rule: If you can’t remember what the file is without opening it, it’s not important enough to keep.

Step 3: Name Things Like You’ll Need to Find Them (20 minutes)

Every file you’re keeping gets renamed. The format is simple:

YYYYMMDD_ProjectName_Description_v01

Example: 20250415_DataStructures_BinarySearchTree_v02.py

Why this works: Six months from now when you need that binary search tree implementation for an interview, you can search “binary search” and actually find it. When you have five projects from the same class, they sort by date automatically. When you have multiple versions, you know which one is most recent.

Do this now while you still remember what everything is. Your future self will thank you.

Go through your most important files, the ones going into Projects and Resources, and rename them. It takes 20 minutes. It saves hours later.

Step 4: Clean Up Your GitHub (25 minutes)

Open your GitHub profile. What does a recruiter see first?

Pin 4-6 repositories. These should be your best work. The project you’re proud of. The one that shows you can code. The one that solves a real problem. Not your intro CS homework. Not the tutorial you followed line-by-line.

Archive the rest. Class assignments from freshman year. Half-finished experiments. The repo where you were just testing something. Go to Settings → Archive this repository. It stays on your profile but signals “this is old, I’m not maintaining it.”

Write one good README. Pick your best project. Write three sections:

  • What it does (2 sentences)

  • How to run it (3-4 bullet points)

  • What you learned or what’s interesting about it (1 paragraph)

Add a screenshot if it has a UI. Add the tech stack at the top: “Built with Python, Flask, PostgreSQL.”

That’s it. One good README is better than five mediocre ones. Start there. Do the others later if you want.

Check your .gitignore. If you’re pushing node_modules or files with API keys, fix it now. Add a .gitignore file. Google “gitignore for [your language]” and copy a standard one.

Step 5: Email and Cloud Storage (20 minutes)

Your email inbox is not a filing system. Your Google Drive is not a dumping ground. Fix both.

Email:

  • Unsubscribe from five newsletters you never read. Right now. Scroll down, click unsubscribe.

  • Create one folder: “Action Required.” Move everything you actually need to respond to into it. Everything else either gets filed into a project folder or deleted.

  • If you’ve been ignoring emails from professors or co-op advisors, respond or file them. Ignoring them doesn’t make them go away.

Cloud Storage (OneDrive/Google Drive):

  • Make the same four folders: Projects, Areas, Resources, Archives.

  • Move your group projects into Archives unless you’re still working on them.

  • Move your co-op application materials (resume, cover letters, company research) into Areas under “Job Search.”

  • Delete duplicate files. If you have “Resume_v1” through “Resume_v8,” keep the current one and delete the rest.

Step 6: Set Up So This Doesn’t Happen Again (10 minutes)

Change one setting: In your browser, go to Settings → Downloads → “Ask where to save each file before downloading.” This forces you to make a decision, Projects, Resources, or Delete, the moment you download something, instead of letting it pile up in Downloads.

Create a “Quick Capture” note. In your phone’s Notes app or Google Keep, make one note called “Quick Capture.” When you have an idea, a link you want to save, or something to remember, it goes there. Once a week, you process it: move it into your actual system or delete it.

Weekly 10-minute review: Every Sunday, spend 10 minutes. Empty your Downloads folder. Process your Quick Capture note. Check if anything in Projects is actually finished and should move to Archives. That’s it. Ten minutes prevents two-hour cleanup sessions.

What This Actually Looks Like

Good enough version (this weekend): Two hours. Make the four folders. Sort this semester’s files. Rename your top five most important projects. Archive half your GitHub repos and write one README. Unsubscribe from five newsletters. Done.

Better version (over spring break): Four hours. Everything above plus: write READMEs for three more projects, fully organize your Google Drive, set up email filters, document what you learned from your best projects while you still remember.

Perfectionist version (don’t do this): Reorganizing every file you’ve ever created, color-coding everything, building a elaborate tagging system. This is procrastination disguised as productivity. Don’t.

Start with good enough. You can always do more later.

The Real Reason to Do This

This isn’t about being organized for the sake of being organized. It’s about not losing your own work. About being able to find the project you’re proud of when a recruiter asks “show me something you’ve built.” About having your resume ready when a co-op opportunity appears with 48 hours to apply. About not spending your winter break trying to remember what you learned last spring.

The file you can’t find might as well not exist. The project you can’t access is gone. The work you did but can’t prove you did doesn’t count.

You put in the work this semester. Don’t lose it in a folder called “Old Stuff” or “Desktop Backup 2025.”

Two hours this weekend. Four folders. That’s the whole system.

Open your file manager. Make the folders. Start sorting.
