# PEDAGOGY — post-build-document

## Concept
Post-build document as evidence of supervisory judgment — five sections that record where the line was drawn between Claude and the teacher.

## Core question
What makes a post-build document an AI-use disclosure rather than a build log?

## Answer
The surface-routing section names human-only tasks and why they stayed human — it records the judgment boundary, not just the output. A build log records what was built; a disclosure records who decided what.

## Teaching pattern
- B00: outcome — five sections, trustworthy workflow
- B01: reframe — supervisory judgment is the deliverable
- B02-B03: generate the document — five sections from the build
- B04: analyze the surface-routing section — palette decisions named as human-only
- B05-B06: score the delegation log — four-dimension completeness check
- B07: timing — write while decisions are fresh
- B08: forward link to dangerous middle activity

## Bloom's level
Applying → Evaluating: students can write a post-build document and distinguish a complete delegation log from an incomplete one.

## Common misconception addressed
Teachers often treat post-build documentation as retrospective self-justification. The video establishes it as a prospective tool — the scoring criteria are known before the build, so the documentation is designed into the workflow, not added after.
