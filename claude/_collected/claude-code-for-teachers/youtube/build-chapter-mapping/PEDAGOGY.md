# PEDAGOGY — build-chapter-mapping

## Concept
Build-to-curriculum mapping — teacher experience as pedagogical content. The teacher's build decisions map to the chapters students are reading.

## Core question
How does the mapping tool benefit teachers who build with Claude Code?

## Answer
It extracts curriculum-relevant moments from the build experience and formats them as classroom activities, so the teacher's implementation becomes Monday's lesson. The gap between theory (student reads chapter) and practice (teacher built the tool) is closed by naming which chapter corresponds to which decision.

## Teaching pattern
- B00: outcome — build decisions mapped to chapters
- B01: reframe — teacher's experience as pedagogical content
- B02-B03: generate the mapping — five decisions, five chapters, five activities
- B04: verify activities are active — write, fill, run, read, measure
- B05-B06: map Act Three — three-file system and deployment verification
- B07: curriculum alignment tool — theory + practice + mapping = Monday's lesson
- B08: series conclusion — full playlist

## Bloom's level
Creating → Teaching: the teacher can generate curriculum-aligned activities directly from build experience without designing activities from scratch.

## Common misconception addressed
Teachers often treat their build experience as separate from their curriculum. The video establishes that every build decision is a curriculum-relevant moment — the mapping tool makes the connection explicit so the teacher can share it with students who are reading the corresponding theory.
