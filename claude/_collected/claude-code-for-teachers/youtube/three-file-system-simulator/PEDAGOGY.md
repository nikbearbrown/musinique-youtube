# PEDAGOGY — three-file-system-simulator

## Concept
Three-file system for classroom tool generation — CLAUDE.md, DESIGN.md, PROJECT.md written before the generation prompt.

## Core question
How do you prevent Claude from choosing your colors and interaction model?

## Answer
Write DESIGN.md and PROJECT.md before the generation prompt. Claude generates inside your constraints, not from training defaults. The three files are the decisions — the generation is the execution.

## Teaching pattern
- B00: contrast — generic prompt produces generic result; three files produce your classroom
- B01: reframe — the three files are the decisions, not the prompt
- B02-B03: show the generation — three files in context, single-file output
- B04: verify the constraints propagated — earth tones, no framework, 44px button
- B05-B06: show constraint propagation — add contrast requirement, watch specific color adjust
- B07: consolidate three roles — technical, aesthetic, pedagogical
- B08: forward link to deployment verification

## Bloom's level
Applying → Creating: students can write three constraint files and generate a classroom tool inside them.

## Common misconception addressed
Teachers often write the prompt first and try to correct Claude's defaults after generation. The video establishes that constraint documents must precede the prompt — Claude generates inside them, not around them.
