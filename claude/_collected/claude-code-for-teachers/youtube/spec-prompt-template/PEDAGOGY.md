# PEDAGOGY — spec-prompt-template

## Concept
Specification prompt template — five slots that force planning before delegation. The planning is the prompt.

## Core question
What does the validation step add to the template generator?

## Answer
It refuses to generate if no specific function or artifact is named — the student cannot skip to the prompt without completing the planning. The vague request is refused at the tool level, not by the teacher.

## Teaching pattern
- B00: contrast — weak prompt produces three wrong functions; template produces one correct function
- B01: reframe — template as bridge between rubric and prompt
- B02-B03: build the template generator — five sections, fill-in placeholders
- B04: analyze the then-run slot — oracle command prevents vague output formats
- B05-B06: add validation — vague request refused, specific request guided
- B07: planning is the prompt — the template works because it refuses shortcuts
- B08: forward link to skill definition

## Bloom's level
Applying → Creating: students can fill the five-element template for any assignment and understand why each slot prevents a category of wrong output.

## Common misconception addressed
Students often believe the prompt is the work. The video establishes that the specification — function signature, invariants, context, output format, negative constraints — is the intellectual work; the prompt is just the delivery mechanism.
