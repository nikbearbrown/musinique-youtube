# PEDAGOGY — grading-skill-definition

## Concept
Reusable skill definitions for repeatable workflows — written once, invoked by name, constrained the same way every time.

## Core question
What makes a skill definition different from a saved prompt?

## Answer
It is invocable by name with typed parameters, stored as a file that can be shared, and constrains the workflow the same way every invocation — including the grade-exclusion constraint that a saved prompt might accidentally omit.

## Teaching pattern
- B00: reach — one skill, this semester and forever, for anyone
- B01: reframe — skill as repeatable workflow vs one-time prompt
- B02-B03: build the definition — YAML frontmatter, workflow description, explicit exclusion
- B04: invoke the skill — consistent output across three submissions
- B05-B06: chain two skills — feedback-drafter takes pattern-analyzer output
- B07: three-element summary — frontmatter, workflow, exclusion
- B08: forward link to build-chapter mapping

## Bloom's level
Applying → Creating: students can write a skill definition and chain two skills without re-specifying the workflow.

## Common misconception addressed
Teachers often save prompts as text files and paste them into each session. The video establishes that skill definitions provide invocation-by-name, typed parameters, and shareable storage — the difference between a copy-paste workflow and a repeatable one.
