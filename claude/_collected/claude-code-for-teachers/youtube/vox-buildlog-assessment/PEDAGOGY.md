# PEDAGOGY — vox-buildlog-assessment
## Why Your Students' CLAUDE.md Is the Best Evidence They Conducted the Build

## One-mechanism rule
This film teaches exactly one mechanism: AI can generate the code but cannot generate the record of the human decisions that governed the code; the build log (specifications, plan-mode edits, handoff conditions, capacity labels, /rewind moments) is the assessment artifact that captures what is irreducibly human in a conducted build, making it the assessment that measures student discipline rather than AI capability.

## Audience
Teachers who assign Claude Code projects to students and need to assess what the student — not the AI — contributed. They may currently grade only the code output. They have not seen a rubric-based system for assessing conducting discipline. They do not need to know rubric design theory — only why the build log is different from the code.

## Learning outcome
After watching, the viewer should understand that: (1) AI can generate code but not the decision record, (2) the build-log rubric measures five conducting dimensions, and (3) formative first, summative at capstone is the recommended deployment.
