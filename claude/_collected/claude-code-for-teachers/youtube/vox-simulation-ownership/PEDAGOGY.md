# PEDAGOGY — vox-simulation-ownership
## Why the Simulation That Took 8 Minutes to Generate Wasn't Theirs

## One-mechanism rule
This film teaches exactly one mechanism: a one-line prompt delegates every design decision — color palette, interaction model, pedagogical scaffolding — to Claude's defaults; the three-file system (CLAUDE.md + DESIGN.md + PROJECT.md) moves those decisions to explicit human choices written before code generation begins, so the resulting simulation belongs to the teacher.

## Audience
Teachers who use Claude Code to build classroom simulations or interactive tools. They have experienced Claude producing technically correct output that doesn't fit their classroom context. They do not need to know Brutalist design theory — only that writing three files before prompting produces output that is theirs.

## Learning outcome
After watching, the viewer should understand that a one-line prompt is not a brief — it is permission to invent — and that the three-file system (45 minutes, before the build) is how a teacher retains ownership of every design decision.
