Chain-of-thought prompting is not about making the model "think longer." It exploits the autoregressive structure of transformers: every output token conditions on all prior tokens, so a wrong intermediate answer cascades into wrong continuations. Adding "think step by step" forces the model to put its work on paper before committing — shifting the probability distribution toward the correct final token.

When it helps: multi-step math, logic puzzles, complex code. When it hurts: simple lookups, yes/no questions, single-fact retrieval. Extended thinking (token budget) is a different mechanism — don't conflate them.

You're not asking Claude to be smarter. You're asking it to structure its probability mass correctly before it has to commit.
