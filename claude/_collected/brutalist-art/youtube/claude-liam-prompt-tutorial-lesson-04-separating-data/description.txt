When your prompt contains user data, that data can contain instructions. Without separation, Claude sees one flat text blob — and may follow an injected command like "ignore previous instructions." The attack surface is the prompt boundary itself.

XML tags are the fix. Wrap your task in `<instructions>`. Wrap the user data in `<document>`. Claude treats tag boundaries as hard separators — text inside `<document>` is data to process, not a command to follow. Multiple documents use indexed tags: `<document index="1">`, `<document index="2">`.

Claude handles XML reliably because it trained on XML-rich web data. The separation isn't a workaround — it's making your prompt structure explicit.
