Token-by-token generation means the first answer token constrains every token after it. If Claude answers before it thinks, that answer is already half-wrong before the reasoning starts.

The fix is one line of XML: a <thinking> block before the <answer> tag. Claude works through the problem inside the scratchpad — intermediate steps, contradictions, corrections — and only writes the final answer once the reasoning is done. The scratchpad externalizes reasoning so it is inspectable and steerable, not just implicit.

This is the precognition principle: make Claude think before it speaks. The rule of thumb — if a human expert would reach for scratch paper, add the thinking tag. Otherwise skip it and save the tokens.
