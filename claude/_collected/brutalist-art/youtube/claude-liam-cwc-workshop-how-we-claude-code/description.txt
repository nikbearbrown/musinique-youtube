Anthropic's own product development workflow has three phases — and they published the exact prompts. Phase 1: Claude interviews you as a user and synthesizes a spec. Phase 2: four divergent static HTML mockups with different visual philosophies. Phase 3: verifiable component architecture — DOM contracts that a test runner can pass or fail.

The surprising part is phase three: you don't audit the code, you audit the verification surface. Run `bun run verify` and each component either passes or fails its spec.

All three phases are reproducible from the published prompts.
