Output format is not cosmetic. It determines what you can do with the response downstream. A JSON response can be parsed. A bullet list cannot be called. A prose summary cannot be queried. Format is architecture.

The prefill technique locks it: put a partial assistant message at the start of the API call. If you want JSON, start Claude's response with an open brace. The model is completing a sequence — it cannot ignore context already written. Prefill is a structural constraint, not a request.

Match format to consumer. JSON for code, markdown for rendered UIs, plain prose for human readers, numbered lists for ordered sequences.
