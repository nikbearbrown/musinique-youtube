# PEDAGOGY — claude-liam-bd-04-reflection-and-ship

## Act structure
8 beats (~367s). Series-closer (E4/4). Cold open (B00) installs the analyze-vs-describe distinction with a concrete contrast pair (same score, two readings). B01 covers Prompt A (distribution analysis, domain property not rubric) and Prompt B (concrete failure with skeptical move). B02 covers Prompt C with the mechanical-execution vs. judgment-about-specification frame and the required "not because" clause. B03 covers Prompt D: the solve-verify asymmetry applied to a specific step with a concrete example of a passing timeline. B04 covers assembly and the Prompt D pre-submit check. BVDT delivers the series-closer verdict, mapping all four prompts to their key rules. BHTF reads the full Prompt D scaffolding prompt aloud and closes with the assembly and submission instruction. BOUT closes.

## Cold open law
Opens on the single distinction that governs the whole episode: analyze versus describe. The contrast is made concrete immediately with two readings of the same score — a describing reading and an analyzing reading. The student knows exactly what they are being asked to do before the first body beat.

## Gap formula
Gap: students have a Boondoggle Score but default to description ("my score has X human steps and Y Claude steps") rather than analysis ("the high Domain Verification count means..."). Each prompt has a distinct failure mode that description falls into. Bridge: four prompts, each with a specific frame (distribution-as-domain-property, concrete-failure, mechanical-execution, solve-verify). Close: one assembled document submitted with confidence that each prompt answers the right question.

## Utility lint
Each prompt has a required sentence or clause named exactly: Prompt A must say what the count reveals about the domain. Prompt B must name the specific wrong output and specific acceptance. Prompt C must contain the "not because" clause. Prompt D must have a number and a cause in the timeline. The Prompt D pre-submit check in B04 is three binary questions — if any answer is no, the student has described the question rather than answered it.

## Vocabulary / register
"Analyze vs. describe" defined in B00 with a paired concrete example. "Mechanical execution" defined in B02 as tasks where the specification fully determines the output. "Judgment-about-specification" defined as the judgment about whether the specification is correct, complete, or applicable — cannot be fully specified in advance. "Solve-verify asymmetry" carried forward from E3 and applied in B03; not re-defined, applied. Register is Pragmatist: imperative, specific, no hedging, direct consequence statements throughout.

## Honesty
The claim that "the grader reads Prompt D last" is the most direct honesty statement in the series — it tells the student where the assessment weight lands without euphemism. The passing Prompt D example (ninety-day discovery at quarterly policy audit) is specific enough to be a real system failure timeline and demonstrates what the requirement actually looks like at full fidelity. The "not because" clause requirement is described honestly as what distinguishes an argued position from a stated preference.

## Length law
8 beats (~367s, ~6:07). The 8-beat structure is warranted: the series-closer must cover four prompts, each with a specific frame and failure mode, plus assembly instructions and the Prompt D scaffolding prompt. B01 covers two prompts (A and B) at 65s — the longest body beat — because both prompts require named failure modes and Prompt B requires both a concrete failure and a connected skeptical move. No beat is padded; each carries a discrete, non-overlapping concept.

## Narration spot-check
B00 contrast pair is concrete: "Domain Verification steps because the claim-flagging domain requires human reviewers to know the policy version in effect on the claim date — that knowledge is not in any document Claude can access." B02 required sentence is stated verbatim: "'I would move [task] from Claude to human because this task requires judgment-about-specification, specifically [what the specification cannot determine], not because [the intuitive reason you initially had].'" B03 passing timeline example has a number (ninety days), a mechanism (quarterly policy audit), and the domain knowledge required (policy-version expertise). BHTF reads the full Prompt D scaffolding prompt verbatim, names all three sub-questions, and instructs the student to write the 150 words themselves after receiving the answers.
