Build a PRD Scope Decision Record with Claude — part of Nina Harris' course Branding and AI.

Every feature list is also a scope commitment. This teardown shows how to generate a PRD that makes the NOT-LIST explicit: the features you decided not to build, and why. We compute a scope_discipline_score (NOT-LIST count / total features) and flag PRDs scoring below 0.25. The worked example: Linear's discipline-first product practice, which reached $35M ARR by 2023.

Prompt used in this video:
  claude "generate a PRD for [PRODUCT DESCRIPTION]:
    Features: [LIST]
    For each feature: one sentence on why this is in scope.
    NOT-LIST: at least 3 features explicitly excluded.
    For each NOT-LIST item: one sentence on why it's out.
    scope_discipline_score = NOT-LIST count / total features.
    Flag if score < 0.25 — add scope discipline warning."
