Two models, same FLOP count, same chip — one runs three times slower. The roofline model splits every model into two regimes: memory-bound (the bus is the constraint) and compute-bound (the silicon is). This video builds the roofline tool with Claude Code and places a FP32 model on it, then shows how int8 quantization shifts the operating point past the ridge.
