# PEDAGOGY — dangerous-middle-detection

## Concept

Dangerous middle detection: identifying hidden supervisory requirements in pattern-looking tasks.

## Question

How do you know if a task that looks like pattern work requires supervision?

## Answer

Map it to the five capacities — any required capacity is a dangerous-middle signal. The contrast case (docstring generation) shows that some tasks are genuinely pure pattern work — the audit takes two minutes and gives you confidence to delegate freely. The signal is not the task type; it is the presence of a required supervisory capacity that the task description doesn't mention.

## Key Insight

The dangerous middle is defined by the gap between what the task description says and what the task actually requires. A GPA sort with tie-breaking looks like a sorting problem; it is actually a Problem Formulation problem (what does alphabetical mean?) and a Plausibility Auditing problem (did we test the tie case?).
