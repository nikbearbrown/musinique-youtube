# PEDAGOGY — five-supervisory-capacities

## Concept

Five supervisory capacities: PF, PA, TO, IJ, EI. What is the supervisory move that prevents the GPA tie-breaking bug from shipping?

## Question

What is the supervisory move that prevents the GPA tie-breaking bug from shipping?

## Answer

Problem Formulation — requiring a tie test case before accepting any output. Without PF, the tie case is never tested. Without the tie case in the test set, the bug is invisible. The move happens before Claude generates, not after.

## Key Insight: Labels as commitments

Naming the five capacities is how you practice them. The label is not the move — it is the commitment to make the move before Claude generates. An unlabeled session runs the same steps but skips the PF requirement because there was no explicit commitment to require it.
